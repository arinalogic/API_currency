# Currency Converter

Консольное приложение на Python для просмотра курсов валют и конвертации сумм.

Проект создан для обучения: показывает, как работать с внешним API, сохранять данные в файл и собирать простое приложение из нескольких модулей.

---

## Назначение проекта

**Currency Converter** — программа, которая:

1. Получает актуальные курсы валют из интернета.
2. Сохраняет ответ в локальный файл, чтобы не запрашивать API при каждом запуске.
3. Показывает курсы выбранных валют в терминале.
4. Конвертирует введённую сумму из одной валюты в другую.

Программа работает в терминале (консоли) и взаимодействует с пользователем через `input()`.

---

## Используемый API

Для курсов валют используется бесплатный публичный API:

**[open.er-api.com](https://open.er-api.com)**

Запрос формируется так:

```
https://open.er-api.com/v6/latest/{base}
```

Где `{base}` — код базовой валюты в формате ISO (например, `USD`, `EUR`, `RUB`).

Пример: `https://open.er-api.com/v6/latest/USD`

API возвращает JSON с полями:

- `base_code` — базовая валюта;
- `time_last_update_utc` — дата последнего обновления;
- `time_next_update_utc` — дата следующего обновления;
- `rates` — словарь курсов.

> Для работы программы нужен доступ в интернет.

---

## Изученные поля ответа API

При изучении документации и анализе ответа метода `latest` были выделены следующие ключевые поля:

| Поле                   | Назначение                                                    |
| ---------------------- | ------------------------------------------------------------- |
| `base_code`            | Код базовой валюты, относительно которой рассчитаны все курсы |
| `time_last_update_utc` | Время последнего обновления курсов                            |
| `time_next_update_utc` | Время следующего обновления данных                            |
| `rates.RUB`            | Курс российского рубля относительно базовой валюты            |
| `rates.USD`            | Курс доллара США относительно базовой валюты                  |

Фрагмент реального ответа API:

```json
{
    "time_last_update_utc": "Sun, 07 Jun 2026 00:02:31 +0000",
    "time_next_update_utc": "Mon, 08 Jun 2026 00:17:01 +0000",
    "base_code": "RUB",
    "rates": {
        "USD": 0.013598,       
        "RUB": 1
        "EUR": 0.011689,

    }
}



   
```

## Возможности программы

### Получение курсов валют

Пользователь вводит код базовой валюты (например, `USD`). Программа загружает курсы и выводит:

- базовую валюту;
- дату последнего обновления;
- курсы **RUB**, **EUR** и **GBP** относительно выбранной базы.

### Кэширование на 24 часа

Ответ API сохраняется в файл `currency_rate.json`.

При следующем запуске программа проверяет:

1. **Существует ли файл** и **не старше ли он 24 часов**.
2. **Совпадает ли** `base_code` в файле с валютой, которую ввёл пользователь.

| Ситуация | Что делает программа |
|----------|----------------------|
| Файл свежий и валюта совпадает | Загружает данные из кэша |
| Файл свежий, но валюта другая | Запрашивает новые данные из API |
| Файла нет или он старше 24 часов | Запрашивает новые данные из API |

### Конвертация сумм

После вывода курсов программа предлагает:

1. Ввести **валюту назначения** (например, `RUB`).
2. Ввести **сумму** для конвертации (например, `100`).

Расчёт выполняется по формуле:

```
результат = сумма × rates[валюта_назначения]
```

Пример вывода:

```
100 USD = 7345.8882 RUB
```

### Обработка ошибок

Программа обрабатывает типичные ошибки и выводит понятные сообщения **без стектрейса** (без длинного технического текста):

| Ошибка | Сообщение |
|--------|-----------|
| Нет интернета или сервер недоступен | Ошибка при получении курсов |
| Неверный код валюты в API | Ошибка при получении курсов |
| Валюта назначения не найдена в `rates` | Валюта не найдена в списке курсов |
| Сумма введена не числом | Сумма должна быть числом |
| Пустой ввод базовой валюты | Код валюты не может быть пустым |

Сообщения выводятся цветом с помощью библиотеки **colorama**.

## Установка и запуск через venv

**Виртуальное окружение (venv)** — изолированная среда Python для проекта. Библиотеки устанавливаются только внутрь проекта, а не в систему.

### 1. Перейти в папку проекта

```powershell
cd путь\к\API_currency
```

### 2. Создать виртуальное окружение

```powershell
python -m venv venv
```

Появится папка `venv`.

### 3. Активировать окружение

**PowerShell (Windows):**

```powershell
.\venv\Scripts\Activate.ps1
```

**CMD (Windows):**

```cmd
venv\Scripts\activate.bat
```

После активации в начале строки терминала появится `(venv)`.

> Если PowerShell блокирует скрипт, выполните один раз:
>
> ```powershell
> Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
> ```

### 4. Установить зависимости

```powershell
pip install -r requirements.txt
```

### 5. Запустить программу

```powershell
python main.py
```

### 6. Выйти из окружения (по желанию)

```powershell
deactivate
```

## Полезно знать

- Файл `currency_rate.json` можно удалить — программа создаст его заново при следующем запросе к API.
- Папку `venv` не нужно копировать другим разработчикам: каждый создаёт своё окружение по инструкции выше.
- Коды валют вводятся латиницей в верхнем регистре: `USD`, `EUR`, `RUB`, `GBP` и т.д.
